# VeriQAI — ENG17 Question Bank

Autogenerated multiple-choice questions for **ENG17: Circuits I** (UC Davis),
built from Anthony Thomas's lecture notes using Claude Code. Questions are
stored in a SQLite database, reviewed through a local web UI, and exported
as Canvas-ready QTI packages.

**Current status:** 108 draft questions across 36 topics (2 modules). All
pending human review.

---

## Quick start — run the review UI

**Mac / Linux**
```bash
cd review_ui
bash start.sh
```

**Windows**
```
Double-click review_ui\start.bat
```

Both scripts install the backend Python dependencies (`fastapi`, `uvicorn`,
`python-dotenv`) and the frontend npm dependencies, start the FastAPI backend
(port 8000) and Vite dev server (port 5173), then open the browser. The only
prerequisites are **Python 3.10+** (with `pip`) and **Node.js 18+**.

If you prefer to start servers manually:
```bash
# Terminal 1 — backend
cd review_ui/backend
pip install -r requirements.txt
uvicorn main:app --reload --port 8000

# Terminal 2 — frontend
cd review_ui/frontend
npm install
npm run dev
```

## Modules

| # | Title | Topics | Questions | Source notes |
|---|-------|--------|-----------|--------------|
| 1 | Response of the Series RLC Circuit | 24 | 72 | `sources/ENG17-series-RLC-derivation/main.tex` |
| 2 | Thévenin & Norton Equivalence + Maximum Power Transfer | 12 | 36 | `sources/ENG17-thevenin-norton-notes/main.tex` |

Module 1 covers: characteristic equation, overdamped / underdamped / critically damped
response, initial conditions from inductor current continuity, amplitude-phase form,
envelope interpretation, worked examples.

Module 2 covers: circuit abstraction, terminal v-i characteristic, Thévenin's theorem,
three R_th methods (short-circuit, source deactivation, test source), Norton derivation,
parameter triangle, maximum power transfer derivation, 50% efficiency.

## Reviewing questions

In the review UI:
- Left sidebar lists chapters grouped by module; badge = questions awaiting review
- Each card shows: question text (KaTeX rendered), 4 choices (correct highlighted green),
  collapsible feedback for both correct and incorrect answers, Bloom level, difficulty dots
- Click **Approve** or **Reject** — decision is written to `veriqa.db` immediately
- Toggle "Show approved" in the sidebar to revisit approved questions

---

## Exporting to Canvas (after review)

```bash
python make_quizzes.py --module 1   # exports approved questions from module 1
python make_quizzes.py --module 2   # exports approved questions from module 2
```

Output zip files land in `exports/`. Import directly into Canvas via
Settings → Import Course Content → QTI .zip file.

---

## Generation pipeline (for reference)

Questions were generated by Claude Code reading the lecture notes in `sources/`
and inserting one topic at a time into `veriqa.db`. The recipe is in `CLAUDE.md`.
